LWJGL Workspace

点击观看 Youtube 上的相应视频。

本篇教程中你将学会如何配置 LWJGL workspace 用于编译与运行 LWJGL 程序。

下载必要文件

从这里下载最新版[1]的 LWJGL 和这里下载最新版的 “slick_util” jar 格式文件。解压 zip 格式文件，将所有的文件与文件夹放在同一目录下。安装 Eclipse ，这里下载 “Classic” 或 “IDE for Java Developers” 版本。将下载的 Eclipse 放在一个你拥有读写权限的目录下（即，不要放在 Program Files 或 Applications 中）。

在 LWJGL workspace 中创建项目

点击运行 Eclipse 。选择工作空间文件夹，其中存放你的所有 Java 项目。点击菜单 “File” ， “New” ， 点击 “Java Project” 并为这个 LWJGL 项目起名。这个项目将存放在先前指定的文件夹内。

添加 library 文件

打开包含了 “src” 和 “bin” 目录的项目目录。添加一个叫作 “lib” 目录，其中包含两个分别叫 “natives” 和 “jars” 的目录。现在的项目目录应包含如下文件夹

- lib
- src
- bin
- lib/natives
- lib/jars

将 “lwjgl.jar” 和 “lwjgl_util.jar” 文件从解压包中与 “slick_util.jar” 文件一同放入 “lib/jars” 目录下。将 zip 文件中适用于当前操作系统的本机文件移动到 “lib/natives” 目录下。

将 jar 文件与本地文件连接

打开 Eclipse 。菜单 “File” 并点击 “Refresh” 。右击先前创建的项目并点击 “Properties” 。进入 “Java Build Path” 并选择 “Libraries” 选项。点击 “Add JARs…” 并选择 “slick_util.jar” ， “lwjgl.jar” ， 和 “lwjgl_util.jar” 文件。展开 lwjgl.jar 表并双击 “Native Library Folder Configuration”。写下你的项目名并在后跟随 “/lib/natives” 然后点击 “OK” 。

测试配置

复制下面代码到你的项目中并运行来测试配置是否成功。如果没有异常并显示一个窗口，则配置正确。

java.lang.UnsatisfiedLinkError: no lwjgl in java.library.path

这个异常将在 LWJGL 无法找到需要的本地文件时被抛出。可能本地文件路径配置错误或选择了其它系统的本地文件。

译注

  1. 此处的 LWJGL 并非为最新版，最新版请到 LWJGL 官网下载。
